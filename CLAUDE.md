## What Is This

Dictum is a native macOS menu bar app for Polish voice dictation.

Pipeline: microphone â†’ WhisperKit STT â†’ optional MLX LLM cleanup â†’ auto-paste. Everything is on-device except HuggingFace queries in `HuggingFaceModelSearch`.

## Build

- Requires Xcode 26+.
- Use Xcode / `xcodebuild`, not `swift build` because MLX Swift compiles Metal shaders.
- Project is generated by XcodeGen: `xcodegen generate` â†’ `open Dictum.xcodeproj`.
- Config lives in `project.yml`.
- App is non-sandboxed by design: CGEvent tap, AXUIElement, pasteboard access, and audio capture.
- Deployment target: macOS 26.0, Apple Silicon only.
- Code signing is manual and requires local configuration.

## Architecture

- This is a layered macOS utility app, not DDD.
- Main flow lives in `DictationPipeline.swift`, which orchestrates recording, transcription, optional LLM cleanup, and paste / clipboard output.
- Main modules: `Audio/`, `Transcription/`, `TextProcessing/`, `ModelBrowser/`, `FloatingIndicator/`, `MenuBar/`, `HotkeyAndPaste/`, `Settings/`.
- `TranscriptionEngine` and `LLMProcessor` are actors.
- Models are loaded lazily and warmed up after load.
- Context mode copies result to clipboard instead of auto-pasting.
- For repo structure, inspect the code directly; keep non-obvious behavior in `findings/`.

## Conventions

- UI language: Polish in `.strings` files only.
- Code, comments, docs, changelog, and commits: English only.
- No transcription history is stored.
- `LSUIElement = true`, so the app has no Dock icon.
- Menu bar icon states are driven from `AppState`.

## Key Gotchas

- Default hotkey is Right Command in modifier-only mode.
- Accessibility permission is mandatory and cannot be auto-granted.
- `resolvePrompt(for:)`: per-app prompt â†’ general prompt â†’ nil. If it resolves to nil, LLM processing is skipped even when cleanup is enabled.
- Launch-at-login fails from DerivedData; it only works from an installed app in `/Applications`.
- Uninstall currently removes the shared MLX cache root, not only Dictum-owned models. See `findings/models.md` and `findings/ui.md`.
- Additional non-obvious behavior belongs in `findings/INDEX.md` and linked finding files.
