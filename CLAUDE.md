# CLAUDE.md

## What is this

Dictum — native macOS menu bar app (Swift/SwiftUI) for voice dictation in Polish. Pipeline: microphone → WhisperKit STT → LLM cleanup (MLX Swift) → auto-paste. 100% on-device.

## Build

- **Requires Xcode 26+** — MLX Swift compiles Metal shaders, `swift build` won't work
- Project generated by XcodeGen: `xcodegen generate` → `open Dictum.xcodeproj`
- Config file: `project.yml`
- Non-sandboxed app — CGEvent tap (system-wide keyboard listener) + AXUIElement (read caret position from any app) + NSPasteboard read/write + AVCaptureDevice audio. Sandbox impossible because CGEvent tap requires Accessibility entitlement unavailable in sandbox. No data leaves the device except HuggingFace API queries in ModelBrowser.
- Deployment target: macOS 26.0, Apple Silicon only
- Code signing: Manual, `CODE_SIGN_IDENTITY: "Dictum Development"`, empty `DEVELOPMENT_TEAM` — requires local configuration

## Dependencies (SPM via XcodeGen)

- `WhisperKit` — `branch: main` (floating pin, unstable API <1.0)
- `mlx-swift-lm` 2.29.3 (exact pin, products: `MLXLLM`, `MLXLMCommon`; `mlx-swift` is a transitive dependency)

## Architecture

### Layers

- **DictumApp.swift** — `@main`, `NSApplicationDelegateAdaptor(AppDelegate.self)` → `AppDelegate` → `MenuBarManager`
- **DictationPipeline.swift** — singleton orchestrator, connects all layers, handles hotkey callbacks; defines `dlog()` → `/tmp/dictum.log`
- **Audio/** — `AudioRecorder` — AVAudioEngine, PCM Float32 16kHz mono, format conversion if hardware ≠ 16kHz
- **Transcription/** — `TranscriptionEngine` (actor), `WhisperModelManager` — lazy model loading
- **TextProcessing/** — `LLMProcessor` (actor) — MLX Swift, `ChatSession` API; `cleanText(rawText:prompt:context:)` — 4 prompt resolution paths: (1) context + `{{context}}` in prompt → both placeholders replaced; (2) context without `{{context}}` → hardcoded EN system prompt, user prompt skipped; (3) no context + `{{text}}` → placeholder replaced; (4) no context, no `{{text}}` → prompt as system message. Automatically strips Qwen3 `<think>...</think>` blocks. Generation params hardcoded: `maxTokens: 2048`, `temperature: 0.7`, `topP: 0.9`
- **ModelBrowser/** — `ModelBrowser` (HuggingFace API, debounce 300ms), `DownloadedModelsManager` (scans `~/Library/Caches/models/mlx-community/`)
- **FloatingIndicator/** — `FloatingIndicatorManager` — NSPanel floating pill at text cursor (AX API), shows recording state + audio level; falls back to mouse position when AX unavailable
- **MenuBar/** — `MenuBarManager` (NSStatusItem + NSPopover), `PopoverView` (~1700 lines, largest file — full UI + onboarding `SetupView`; `SetupView` gate: `isSetupComplete = allGranted && downloadedModelIds.contains(sttModelId)`, 3 steps: permissions → STT download → optional LLM download; `mainContent` sections: Header, RecordingSettings, STTModel, LLMModel (only shown when `llmCleanupEnabled = true`; contains prompt + per-app prompts), DownloadedModels, Footer), `MenuBarIcon` (NSImage factory per AppState — template icon + custom recording icon)
- **HotkeyAndPaste/** — `GlobalHotkeyManager` (CGEvent tap, two modes: modifier-only and key+modifier; Escape cancels recording via `cancelHandler`), `SelectedTextReader` (synchronously captures selected text via simulated Cmd+C in event tap callback, result → `DictationPipeline.pendingSelectedContext`), `PasteManager` (NSPasteboard + CGEvent Cmd+V, save/restore clipboard)
- **Settings/** — `AppSettings` singleton (`@AppStorage` + `@Published` state), `PermissionsManager` singleton (AX + Microphone polling), `AppState` enum

### Key patterns

- `TranscriptionEngine` and `LLMProcessor` are **Swift actors** (thread safety)
- Models loaded **lazily** — on first recording, not at app launch
- `AppSettings.shared` — singleton with `@AppStorage` + `@Published` state; `AppState` enum (idle, recording, transcribing, processingLLM, done (defined but not currently set — pipeline transitions directly to idle after paste), error(String) — value-carrying case with associated error message)
- `DictationPipeline.shared` — singleton orchestrator, two modes: normal (auto-paste) and context (clipboard only)
- CGEvent tap requires Accessibility permission — `AXIsProcessTrusted()` (system prompt via `AXIsProcessTrustedWithOptions`, but auto-grant impossible)
- **Hotkey**: default Right ⌘ (keyCode 54) in modifier-only mode; alternatively key+modifier combo; Escape cancels recording

### Data flow

```
Hotkey (GlobalHotkeyManager)
  → SelectedTextReader.readSelectedText() → pendingSelectedContext  [optional, synchronous in event tap]
  → DictationPipeline.startRecording()
    → AudioRecorder.startRecording()
    → AudioRecorder.stopRecording() → [Float]
    → TranscriptionEngine.transcribe() → String (raw)
    → LLMProcessor.cleanText(rawText, prompt, context: selectedContext?) → String (clean)  [optional]
    → [if context] NSPasteboard.general (clipboard only, user pastes manually)
    → [else]       PasteManager.pasteText() → Cmd+V (auto-paste)
```

## Conventions

- UI language: Polish (localized .strings)
- Code, comments, documentation, changelogs, commit messages: **English only** — Polish appears only in UI strings (.strings files)
- No Polish in CHANGELOG.md, README.md, CLAUDE.md, FINDINGS.md, or any non-UI file
- No transcription history — app does not store user data
- `LSUIElement = true` — app does not appear in Dock
- Menu bar icon states: idle (template mic.fill), recording (custom icon with red dot REC), other states use template icon (no separate colors for transcribing/processingLLM/done)

## Dev cycle

- **After every code change**: `pkill -x Dictum` → `xcodebuild` → run new version. Always kill → build → launch.
- App is `LSUIElement` — no Dock icon, `pkill -x Dictum` is the only way to kill it
- Launch-at-login (`SMAppService.mainApp`) silently fails from DerivedData — requires installed copy in `/Applications`

## Debugging

- `dlog()` → `tail -f /tmp/dictum.log` — all pipeline logs (~45 callsites)

## Release

Use `/release` skill (`.claude/skills/release/SKILL.md`) to automate version bump, CHANGELOG update, tag, and GitHub release push. Manual process: bump `MARKETING_VERSION` in `project.yml` → update CHANGELOG.md → `git tag v*` → `git push origin main --follow-tags`. The `release.yml` workflow builds `.app` on `macos-26` runner and creates a GitHub Release with `Dictum.zip`.

## Known limitations

See [README.md](README.md#known-limitations) for the full list. Additional developer-relevant limitations:

- MLX Swift requires xcodebuild (Metal shaders)
- Accessibility permission — system prompt via `AXIsProcessTrustedWithOptions`, but auto-grant impossible; user confirms manually
- LLM output capped at 2048 tokens (`maxTokens` hardcoded in `LLMProcessor`) — long dictations may be silently truncated

See [FINDINGS.md](FINDINGS.md) for non-obvious gotchas and workarounds discovered during development.
