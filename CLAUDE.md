# CLAUDE.md

## What is this

Dictum — native macOS menu bar app (Swift/SwiftUI) for voice dictation in Polish. Pipeline: microphone → WhisperKit STT → LLM cleanup (MLX Swift) → auto-paste. 100% on-device.

## Build

- **Requires Xcode 26+** — MLX Swift compiles Metal shaders, `swift build` won't work
- Project generated by XcodeGen: `xcodegen generate` → `open Dictum.xcodeproj`
- Config file: `project.yml` (`xcodeVersion: "16.0"` is XcodeGen's min-Xcode-to-open setting, not the build requirement)
- Non-sandboxed app — CGEvent tap (system-wide keyboard listener) + AXUIElement (read caret position from any app) + NSPasteboard read/write + AVCaptureDevice audio + ScreenCaptureKit (screenshot for VLM context). Sandbox impossible because CGEvent tap requires Accessibility entitlement unavailable in sandbox. No data leaves the device except HuggingFace API queries in ModelBrowser.
- Deployment target: macOS 26.0, Apple Silicon only
- Code signing: Manual, `CODE_SIGN_IDENTITY: "Dictum Development"`, empty `DEVELOPMENT_TEAM` — requires local configuration

## Dependencies (SPM via XcodeGen)

- `WhisperKit` — `exactVersion: 0.18.0` (unstable API <1.0)
- `mlx-swift-lm` — `exactVersion: 3.31.3` (products: `MLXLLM`, `MLXVLM`, `MLXLMCommon`; `mlx-swift` is a transitive dependency). `MLXVLM` enables runtime VLM dispatch — `loadModelContainer` auto-detects vision models via `ModelFactoryRegistry`
- `swift-transformers` — `minorVersion: "1.1.6"` (products: `Hub`, `Tokenizers`; bridges HuggingFace Hub downloads and tokenizer loading into MLXLMCommon via `HubDownloader`/`TransformersTokenizerLoader` adapters in `LLMProcessor.swift`)
- `Sparkle` — `from: "2.7.0"` (auto-updates via appcast from GitHub Releases)

## Architecture

### Layers

- **DictumApp.swift** — `@main`, `NSApplicationDelegateAdaptor(AppDelegate.self)` → `AppDelegate` → `MenuBarManager`
- **DictationPipeline.swift** — singleton orchestrator, connects all layers, handles hotkey callbacks; defines `dlog()` → `~/Library/Logs/Dictum/dictum.log`
- **Audio/** — `AudioRecorder` — AVAudioEngine, PCM Float32 16kHz mono, format conversion if hardware ≠ 16kHz
- **Transcription/** — `TranscriptionEngine` (actor), `WhisperModelManager` — lazy model loading
- **Context/** — `ContextGatherer` (gathers all enabled sources in parallel via `ContextOptions`: screenshot, selected text, clipboard), `ScreenshotCapture` (ScreenCaptureKit `SCScreenshotManager`, captures frontmost window, downscales to max 1024px longest side for VLM), `DictationContext` (Sendable value type carrying appName, bundleId, selectedText, screenshot, clipboardText, clipboardImage)
- **TextProcessing/** — `LLMProcessor` (actor) — MLX Swift, `ChatSession` API; runtime VLM detection (`isVisionModel = context.model is VLMModel`). `cleanText(rawText:prompt:context:)` — if context provided: system prompt from `resolvePrompt`, user message = structured parts (App name, Selected text, Clipboard text, Spoken words) + images via multi-image `respond(to:images:videos:)` API; if no context: prompt as system message, rawText as user message. Images (screenshot, clipboard) only passed when `isVisionModel == true`, silently skipped for text-only models. Automatically strips Qwen3 `<think>...</think>` blocks. Generation params hardcoded: `maxTokens: 2048`, `temperature: 0.7`, `topP: 0.9`
- **ModelBrowser/** — `ModelBrowser` (HuggingFace API, debounce 300ms), `DownloadedModelsManager` (scans `~/Library/Caches/models/mlx-community/`)
- **FloatingIndicator/** — `FloatingIndicatorManager` — NSPanel floating pill at text cursor (AX API), shows recording state + audio level; falls back to mouse position when AX unavailable
- **MenuBar/** — `MenuBarManager` (NSStatusItem + NSPopover), `PopoverView` (main container, routes to `SetupView` / `PermissionsNeededView` / main content via `isSetupComplete` and `hasCompletedSetup`), `SetupView` (onboarding: permissions → STT download → optional LLM download; gate: `isSetupComplete = allGranted && downloadedModelIds.contains(sttModelId)`), `PermissionsNeededView` (post-update recovery when permissions are lost), `LLMModelSection`, `STTModelSection`, `STTLanguageSection`, `DownloadedModelsSection`, `PromptTextEditor`, `InstalledAppPickerSheet`, `MenuBarIcon` (NSImage factory per AppState — template icon + custom recording icon)
- **HotkeyAndPaste/** — `GlobalHotkeyManager` (CGEvent tap, two modes: modifier-only and key+modifier; Escape cancels recording via `cancelHandler`), `SelectedTextReader` (synchronously captures selected text via simulated Cmd+C in event tap callback, result → `DictationPipeline.pendingSelectedContext`), `PasteManager` (NSPasteboard + CGEvent Cmd+V, save/restore clipboard)
- **Settings/** — `AppSettings` singleton (`@AppStorage` + `@Published` state), `PermissionsManager` singleton (AX + Microphone polling), `UpdaterManager` singleton (Sparkle `SPUStandardUpdaterController`, auto-update checks on launch, injected into PopoverView), `AppState` enum
- **Resources/** — `Info.plist` (contains `SUPublicEDKey` for Sparkle), `Dictum.entitlements` (non-sandboxed, no `com.apple.security.app-sandbox`), `PrivacyInfo.xcprivacy` (App Store privacy manifest), `en.lproj/` + `pl.lproj/` localization `.strings`

### Key patterns

- `TranscriptionEngine` and `LLMProcessor` are **Swift actors** (thread safety)
- Models loaded **lazily** — on first recording, not at app launch
- `AppSettings.shared` — singleton with `@AppStorage` + `@Published` state; `AppState` enum (idle, warmingUp, recording, transcribing, processingLLM, done (handled in icon/UI but currently never assigned — pipeline goes directly to idle after paste), error(String) — value-carrying case with associated error message)
- `resolvePrompt(for:)` — always returns a non-empty string. Base = `unifiedSystemPrompt` (or `defaultUnifiedPrompt` if empty); then: (1) if per-app prompt exists and enabled → append as "Additional instructions for this app"; (2) else if `llmGeneralPromptEnabled` && `llmPrompt` non-empty → append as "Additional instructions". Never returns nil — LLM always runs when `llmCleanupEnabled = true`
- `resolveSTTLanguage(for:)` priority: (1) enabled per-app STT language for frontmost bundle ID → (2) general `sttLanguage` setting → returns Whisper code or nil (auto-detect)
- `DictationPipeline.shared` — singleton orchestrator. Paste mode determined at output time: if `context?.selectedText != nil` → clipboard only (user pastes manually); else → `PasteManager` auto-paste (Cmd+V). Disabling `contextSelectedText` toggle forces auto-paste even when text is selected
- CGEvent tap requires Accessibility permission — `AXIsProcessTrusted()` (system prompt via `AXIsProcessTrustedWithOptions`, but auto-grant impossible)
- **Hotkey**: default Right ⌘ (keyCode 54) in modifier-only mode; alternatively key+modifier combo; Escape cancels recording

### Data flow

```
Hotkey (GlobalHotkeyManager)
  → SelectedTextReader.readSelectedText() → pendingSelectedContext  [optional, synchronous in event tap]
  → DictationPipeline.startRecording()
    → AudioRecorder.startRecording()
    → AudioRecorder.stopRecording() → [Float]
    → TranscriptionEngine.transcribe() → String (raw)
    → [if smartContextEnabled && any source toggle on]
        ContextGatherer.gather(selectedText, frontmostApp, options) → DictationContext
          (parallel: ScreenshotCapture + clipboard read, respects per-source toggles)
    → [if llmCleanupEnabled]
        LLMProcessor.cleanText(rawText, prompt, context?) → String (clean)
          (images passed to VLM via multi-image API; skipped for text-only LLM)
    → [if context.selectedText != nil] NSPasteboard.general (clipboard only, user pastes manually)
    → [else]                           PasteManager.pasteText() → Cmd+V (auto-paste)
```

## Conventions

- UI language: Polish (localized .strings)
- Code, comments, documentation, changelogs, commit messages: **English only** — Polish appears only in UI strings (.strings files)
- No Polish in CHANGELOG.md, README.md, CLAUDE.md, findings/, or any non-UI file
- No transcription history — app does not store user data
- `LSUIElement = true` — app does not appear in Dock
- Menu bar icon states: idle (template — rounded rect with mic.fill cutout), recording (custom icon with red dot REC), other states (warmingUp/transcribing/processingLLM/done) use template icon

## Dev cycle

- **After every code change**: `pkill -x Dictum` → `xcodebuild -project Dictum.xcodeproj -scheme Dictum` → run new version. Always kill → build → launch.
- No PR/push CI — only the tag-triggered `release.yml` (and `pages.yml` which triggers on push to `main` when `website/**` changes). Validate locally with `xcodebuild` before merging.
- App is `LSUIElement` — no Dock icon, `pkill -x Dictum` is the only way to kill it
- Launch-at-login (`SMAppService.mainApp`) silently fails from DerivedData — requires installed copy in `/Applications`

## Debugging

- `dlog()` → `tail -f ~/Library/Logs/Dictum/dictum.log` — all pipeline logs (~60 callsites across 10 files)

## Known limitations

- MLX Swift requires xcodebuild (Metal shaders) — `swift build` will not work
- Accessibility permission — system prompt via `AXIsProcessTrustedWithOptions`, but auto-grant impossible; user confirms manually
- Screen Recording permission is optional for setup gate (`allGranted` only checks AX + mic) but required for screenshot context — if denied, `ScreenshotCapture.captureFrontmostWindow()` returns nil silently
- LLM output capped at 2048 tokens (`maxTokens` hardcoded in `LLMProcessor.cleanText()`) — long dictations may be silently truncated
- When a text-only LLM is loaded (`isVisionModel == false`), screenshots and clipboard images are silently discarded — only text context (app name, selected text, clipboard text, spoken words) is submitted to the model
- Context mode (triggered when `context?.selectedText != nil`) replaces clipboard contents with LLM output; original clipboard is not restored. `PasteManager`'s save/restore logic is only used in the auto-paste (normal) path
- First WhisperKit model run triggers CoreML compilation on ANE (~30-60s)
- Floating indicator falls back to mouse position when the app doesn't expose AX text cursor (Electron, terminals)
- Uninstall (`performUninstall()` in `PopoverView`) deletes `~/Library/Caches/models/` — the entire shared MLX cache root, not just Dictum's `mlx-community/` subdirectory. Other MLX apps' models may be affected
